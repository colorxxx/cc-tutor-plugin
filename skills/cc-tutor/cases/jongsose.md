# 사례: 종소세-안내문 스킬 — 템플릿 생성에 python 검증 모듈을 붙여 "틀리면 저장 안 함"을 강제하다

> 배우는 패턴: (c) 템플릿 생성 + (d) python 검증 모듈  |  난이도: ★★★★

## 1. 이 스킬이 푸는 실무 문제

종합소득세 시즌이면 수십 개 거래처의 신고서 PDF를 읽어 안내문 보고서(docx)를 만들어야 한다. 손으로 하면 매출·세액 숫자를 옮기다 오타가 나고, 그 보고서가 그대로 거래처에 나간다. 이 스킬은 노션에서 "안내문 생성요청" 상태인 거래처만 골라 → 신고서 PDF 파싱 → docx 4섹션 생성 → **신고서 금액과 보고서 금액이 맞는지 python으로 검증** → 통과해야만 저장하고 노션 상태를 "생성완료"로 바꾼다. 검증이 핵심 안전장치다.

## 2. SKILL.md 핵심 발췌

frontmatter의 description이 워크플로 전체를 한 문장으로 압축하고 트리거까지 정의한다.

```yaml
---
name: 종소세-안내문
description: 노션 정기신고내역 DB에서 진행상태 "안내문 생성요청"인 거래처를
  추출 → 신고서 PDF 파싱 → 보고서 docx 생성 → 신고서 금액 검증 →
  쉐어포인트 저장 → 노션 상태 "안내문 생성완료" 변경. 트리거 -
  "/종소세-안내문", "신규 완료 거래처 안내문", "[거래처명] 종소세 안내문".
---
```

검증 단계가 이 스킬을 단순 생성기와 갈라놓는다 — 틀리면 멈춘다.

```markdown
### 6단계: 검증 (저장 전 필수)
verifier.py 의 verify_amounts() 호출.
docx에서 텍스트 추출 후 핵심 금액이 모두 포함됐는지 확인:
- 매출액, 필요경비, 사업소득금액, 과세표준, 산출세액, 기납부세액, 최종납부세액

검증 실패 시:
- 저장하지 않음
- 노션 상태 변경하지 않음
- 사용자에게 "어떤 항목 불일치" 보고
```

덮어쓰기·중복 호출을 막는 안전장치도 명세에 박혀 있다.

```markdown
## 안전장치 (절대 위반 금지)
1. 기존 파일 덮어쓰기 절대 금지 — _next_versioned_path()가 빈 버전을 찾아
   v3.0 → v3.1 → v3.2 자동 증가. 이전 안내문은 항상 보존.
2. "안내문 생성요청" 상태인 거래처만 처리 (다른 상태는 자동 스킵)
7. 중복 호출 안전성 — 이미 "생성완료"면 자동 스킵
```

## 3. 구조

이 스킬은 SKILL.md가 오케스트레이션을 서술하고, 실제 작업은 python 모듈들이 나눠 맡는다.

```
skills/종소세-안내문/
├── SKILL.md            # 9단계 워크플로 + 안전장치 서술
├── engine.py           # 메인 엔진 (워크플로 조립, 버전 경로 관리)
├── parser.py           # PDF 파싱 + "가공경비" 키워드 파싱
├── generator.py        # docx 4섹션 생성 (회계/세금/Comment/수수료)
├── verifier.py         # 보고서 ↔ 신고서 금액 대조 검증
├── notion_sync.py      # 노션 fetch/update 래퍼 (상태 변경)
└── manual_data.json    # 이미지 PDF 등 파싱 불가 건 수동 오버라이드

흐름:
노션 추출 → PDF 위치확인 → parser 파싱 → generator docx생성
   → verifier 검증 ─실패→ 저장X·상태변경X·보고
                    └성공→ 버전경로 저장 → notion_sync 상태변경 → 보고
```

## 4. 여기서 배울 패턴 3가지

1. **생성과 검증을 분리하고, 검증을 게이트로 둔다.** 보고서를 만든 뒤 `verify_amounts()`가 신고서 원본 숫자와 대조해 하나라도 안 맞으면 저장도 노션 변경도 안 한다. LLM이 만든 결과물을 그대로 믿지 않고, 결정론적 python으로 한 번 더 거른다 — 돈 숫자를 다루는 업무의 필수 안전장치다.

2. **무거운 로직은 SKILL.md가 아니라 python 모듈로 뺀다.** PDF 파싱 정규식, 가공경비 키워드 추출, 버전 경로 계산 같은 복잡하고 결정론적이어야 할 작업은 `parser.py`/`engine.py`에 둔다. SKILL.md는 "언제 무엇을 호출할지"만 서술해, 모델이 매번 같은 코드를 재발명하지 않게 한다.

3. **부수효과(저장·외부 상태변경)에 멱등성을 건다.** 덮어쓰기 금지 + 버전 자동 증가 + "생성완료면 스킵" 덕에 같은 명령을 두 번 돌려도 기존 파일이 안 깨지고 중복 처리도 안 된다. 자동화가 실수로 두 번 실행돼도 안전하도록 설계한 점이 실무적이다.
