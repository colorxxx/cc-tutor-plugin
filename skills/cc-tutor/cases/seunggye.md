# 사례: 승계 스킬 — references/ 번들로 20년차 전문가의 암묵지를 주입하다

> 배우는 패턴: (b) references/ 번들로 암묵지 주입  |  난이도: ★★★

## 1. 이 스킬이 푸는 실무 문제

가업승계·구조조정(합병/분할/현물출자/주식교환) 자문은 상증세법·조특법·법인세법이 얽혀 있어, 일반 LLM이 답하면 조문 번호를 헷갈리거나 사후관리 요건을 빠뜨린다. 이 스킬은 ○○회계법인이 직접 정리한 핵심 정리·예규/판례·전략 시나리오를 `references/` 폴더에 PDF/MD로 번들해 두고, 모델이 **먼저 로컬 자료를 Grep으로 뒤진 뒤** 부족할 때만 외부(NotebookLM)를 조회하게 한다. 즉, 사무소의 축적된 암묵지를 모델 컨텍스트로 끌어오는 구조다.

## 2. SKILL.md 핵심 발췌

frontmatter가 description에 "1차 로컬 references + 2차 NotebookLM 심화"라는 설계 의도를 못 박는다.

```yaml
---
name: 승계
description: "가업승계, 구조조정 전문 자문 에이전트. /승계 로 호출.
  PDF 기반 로컬 references 1차 참조 + NotebookLM 2차 심화 조회."
---
```

워크플로의 2단계가 이 패턴의 핵심 — 로컬 자료를 1차 소스로 삼는다.

```markdown
### 2단계: 로컬 references 검색 (1차)
경로: {문서_경로}/references/
- 가업승계_핵심정리.md (요건, 사후관리, 증여세 과세특례)
- 구조조정_핵심정리.md (합병/분할/현물출자/주식교환)
- 법령_조문_요약.md (상증세법, 조특법, 법인세법 조문)
- 예규_판례_정리.md (가업승계 100건+, 구조조정 다수)
- 승계전략_시나리오.md (유형별 로드맵, 10/20년 시뮬레이션)
- 실무쟁점_심화.md (사업무관자산, 고용유지, 평가, 유류분)
- Grep으로 키워드 매칭하여 관련 내용 확인

### 3단계: NotebookLM 심화 조회 (2차, 비동기)
NotebookLM 조회 전, 로컬 references로 충분히 답변 가능하면 이 단계를 생략한다.
```

답변 형식도 명세에 박아 둔다 — 보수적 입장 / 공격적 입장을 나누고, 의사결정용 비교표까지.

```markdown
3. 보수적 입장 — 근거(법령/예규/판례), 효과, 장점, 단점
4. 공격적 입장 — 근거, 효과, 장점, 단점
5. 선택지 비교표 (10년/20년 관점, 세부담·사후관리 리스크·경영권 안정성)
```

## 3. 구조

```
skills/승계/
├── SKILL.md
└── references/                       # 사무소가 직접 큐레이션한 지식 번들
    ├── 가업승계_핵심정리.md   (약 18KB)
    ├── 구조조정_핵심정리.md   (약 23KB)
    ├── 법령_조문_요약.md      (약 10KB)
    ├── 예규_판례_정리.md      (약 21KB)
    ├── 승계전략_시나리오.md   (약 45KB)
    └── 실무쟁점_심화.md       (약 52KB)

질문 → [1차] references Grep → 충분하면 답변
                            └ 부족하면 → [2차] NotebookLM 비동기 조회 → 답변
```

## 4. 여기서 배울 패턴 3가지

1. **암묵지를 references/ 파일로 떨어뜨려 컨텍스트화한다.** "20년차 회계사가 아는 것"을 프롬프트에 다 적을 수는 없다. 대신 사무소가 정리한 자료를 폴더에 두고 Grep으로 필요한 부분만 끌어오면, 모델이 사내 기준에 맞춰 답한다. 약 170KB의 도메인 지식을 통째로 컨텍스트에 넣지 않고 검색으로 골라 쓴다.

2. **1차 로컬 / 2차 외부의 계층(fallback) 설계.** 로컬 references로 충분하면 NotebookLM을 생략한다. 비싸고 느린 외부 조회를 기본으로 돌리지 않고, 로컬이 못 메우는 심화 질문에만 쓴다. 비용·속도·신뢰도를 단계로 나눈 것.

3. **출력 형식을 강하게 규격화한다.** "보수적/공격적 입장 + 10/20년 비교표 + 실무 의견"을 못 박아, 모델이 자유 형식으로 흐르지 않고 의사결정에 바로 쓰는 문서를 낸다. 톤("~임. ~판단됨." 체, 이모지 금지)까지 지정해 사무소 문체를 통일했다.

## 5. 토론 질문

1. 여러분 분야에서 "신입은 모르지만 베테랑은 아는" 암묵지는 무엇인가요? 그걸 references 파일 몇 개로 쪼갠다면 어떤 제목들이 될까요?
2. 이 스킬은 references가 stale(낡음)해질 위험이 있습니다. 법이 개정되면 로컬 파일이 틀려지죠. 1차 로컬 / 2차 외부 중 어느 쪽을 더 신뢰하도록 설계해야 할까요? 그 판단 기준은?
