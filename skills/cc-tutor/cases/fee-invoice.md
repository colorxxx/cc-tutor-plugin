# 사례: 수수료 청구서 스킬 — docx 템플릿에 값만 갈아끼우는 가장 단순한 자동화

> 배우는 패턴: (a) docx 템플릿 치환  |  난이도: ★

## 1. 이 스킬이 푸는 실무 문제

○○회계법인은 법인세 신고가 끝날 때마다 거래처에 수수료 청구서(docx)를 보낸다. 매번 같은 양식에 거래처명·금액·날짜만 바뀌는데, 사람이 손으로 채우면 부가세 계산을 틀리거나 서식이 어긋난다. 이 스킬은 잘 만들어 둔 `template.docx`를 그대로 열어 셀 텍스트만 교체하고, 부가세·합계·납부기한은 자동 계산한다. "새 문서를 만들지 않고 템플릿을 채운다"가 핵심이다.

## 2. SKILL.md 핵심 발췌

이 스킬에는 frontmatter가 따로 없고, 본문이 곧 작업 명세서다. 가장 교육적인 두 부분 — 입력값 정의와 치환 규칙.

```markdown
## 작성 전 필수 입력값
| 항목 | 설명 | 필수 |
| 거래처명 | 수신처 이름 | O |
| 귀속연도 | 결산 대상 사업연도 (예: 2025) | O |
| 공급가액 | 수수료 원래 금액 | O |
| 할인가격 | 할인 금액 (없으면 0) | - |
| 기납부액 | 이미 납부한 금액 (없으면 0) | - |

### 자동 계산 항목
- 청구액 = 공급가액 - 할인가격
- 부가가치세 = 청구액 x 10%
- 합계 = 청구액 + 부가가치세
- 납부기한 = 작성일 + 7일
```

치환은 "어느 테이블, 몇 번째 행"을 좌표로 콕 집어 지정한다.

```markdown
### 치환 대상 — Table 1 (수수료 내역)
| Row | 치환 내용 | 서식 |
| 1 | 공급가액 | 우측정렬, 천단위 콤마 + "원" |
| 2 | 할인가격 | 0이면 "-" 표시 |
| 5 | 합계   | 천단위 콤마 + "원", bold |
```

생성 로직은 python-docx로 템플릿을 열어 값을 꽂는 함수 하나로 끝난다.

```python
from docx import Document
def create_fee_invoice(client_name, fiscal_year, supply_amount, discount=0, ...):
    billed = supply_amount - discount      # 청구액
    vat = int(billed * 0.1)                # 부가세
    total = billed + vat                   # 합계
    doc = Document("template.docx")        # 새로 만들지 않고 템플릿을 연다
    # Table 0/1/2, 본문 paragraph 치환
    return doc
```

저장 경로·계좌번호·브랜드 색상(`#2D5050`) 같은 고정값은 SKILL.md에 못 박아 둔다. 경로는 `{문서_경로}`처럼 사내 공유 드라이브를 가리킨다.

## 3. 구조

```
skills/fee-invoice/
├── SKILL.md        # 입력값·치환 좌표·서식 규칙·계산식
└── template.docx   # 손대지 않는 원본 양식 (서식의 단일 출처)

흐름:
입력 수집 → 금액 자동계산 → template.docx 열기 → 셀 치환
        → 파일명 규칙대로 저장 → (선택) 노션 안내 페이지 생성
```

## 4. 여기서 배울 패턴 3가지

1. **템플릿이 곧 서식의 단일 출처(Single Source of Truth)다.** 폰트·색·테두리를 코드로 재현하려 하지 말고, 디자인이 끝난 docx를 열어 텍스트만 바꾼다. "template.docx 서식을 절대 변경하지 않는다"는 규칙 한 줄이 결과물 품질을 보장한다.

2. **치환 위치를 좌표로 명세한다.** "Table 1, Row 5에 합계, bold"처럼 위치를 못 박으면 모델이 헤매지 않는다. 자연어로 "적당한 곳에 합계를 넣어줘"라고 하는 것과 정반대 — 반복 업무일수록 좌표화가 안정성을 키운다.

3. **계산 규칙과 예외를 SKILL.md가 직접 정의한다.** 부가세 10%, 납부기한 +7일, "0이면 0원 대신 '-'" 같은 도메인 규칙을 문서에 박아 두면, 모델은 매번 추론하지 않고 그대로 따른다. 실무 규칙을 코드 주석이 아니라 스킬 명세에 둔 점이 포인트다.
