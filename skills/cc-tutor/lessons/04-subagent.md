# L4. Subagent로 검증 맡기기

## 이 레슨의 목표
- "검증은 별도 AI가 한다"를 **컨텍스트 관점에서 구체적으로** 이해한다 (추상적 설명 금지)
- Subagent를 호출하는 스킬(Task A→B→C 패턴)을 **직접 만들어본다**

---

## 개념: 왜 별도 AI인가 — L2 복습에서 출발

L2에서 배운 것 두 가지를 다시 꺼낸다:

**① AI는 매번 대화 전체를 다시 읽는다**
→ 한 세션에서 오래 일한 AI의 작업대에는 **자기가 내린 결론들이 잔뜩 쌓여 있다.**
→ 그 결론을 계속 다시 읽으면서 일하니까, **틀린 결론도 "일관되게" 유지**하려 한다. (사람의 확증편향과 같은 구조)

**② Lost in the Middle**
→ 긴 세션의 중간에 있던 중요한 규칙을 이미 놓치고 있을 수도 있다.

그래서 결론:
> **검증은 깨끗한 작업대를 가진 새 AI(subagent)에게 맡긴다.**
> - 새 AI는 이전 대화를 모른다 = 앵커링(물듦) 없음
> - 결과물만 받아서 "이거 맞아?"를 객관적으로 본다
> - 사람으로 치면: 내가 쓴 보고서를 **옆자리 동료에게 검토** 받는 것

비유 정리: Skill=레시피, MCP=주방 도구, **Agent=요리사**. Subagent는 "검수 담당 요리사를 한 명 더 부르는 것."

---

## 튜터의 질문 (개념 확인)

1. "오래 일한 AI에게 '네가 한 거 검토해봐'라고 하면 뭐가 문제일까요?"
   (답: 자기 결론을 다시 읽으며 일하니까 자기 실수를 그대로 통과시키기 쉬움)
2. "고객명이 빠진 안내문을 subagent에게 검증시키면 — 통과? 지적? 지어냄?"
   (답: 규칙에 '4가지 확인'이 적혀 있으면 지적함. **검증 기준도 글로 적어야 작동**)

## 🔨 실습 (메인): Subagent 호출 스킬 직접 만들기

> 튜터 지침: **대신 만들어주지 말 것.** 아래 골격을 보여주고, 학습자가 자기 업무로 [입력]/[할일]/[결과물]을 채우게 한다. 막히면 힌트만.

핵심 패턴 = **Task A (내가 한다) → Task B (subagent가 검증) → Task C (결과 반영, 미진하면 A부터 재시도)**

```markdown
---
name: my-skill
description: ~~할 때 쓰는 skill
---

# My Skill

## Task A: 먼저 직접 하는 일
[입력] ~~를 읽어
[할일] ~~를 정리해
[결과물] 결과를 md파일로 저장해 → TaskA_output.md

## Task B: 다 끝나면 subagent 호출
Task A가 끝나면, 아래 형식으로 subagent를 띄워서 검증을 시킨다:

    Agent(
      subagent_type: "general-purpose",
      description: "B 작업 처리",
      prompt: |
        [배경] Task A에서 이런 결과가 나왔다: <Task A 결과 요약>
        참조 파일: TaskA_output.md
        [할 일] 이 결과에 틀린 값이 있는지 검토해라
        [결과물] TaskB_output.md
    )

## Task C: subagent 결과 받아서 마무리
subagent가 리턴한 요약을 받아서:
1. 지적사항을 결과물에 반영해
2. 사용자에게 최종 보고
3. 만약 결과가 미진하다면 Task A부터 다시 시작.
```

### 실습 진행 순서
1. 학습자의 업무 하나 고르기 (예: 거래처 자료요청 안내문, 메모 정리 등)
2. Task A의 [입력]/[할일]/[결과물]을 학습자가 직접 채우게 한다
3. Task B의 "검증 기준"을 학습자가 한 줄 적게 한다 (예: "금액에 천단위 쉼표 있는지, 마감일 형식 맞는지")
4. `~/.claude/skills/` 아래 저장 후 발동 → **subagent가 실제로 뜨는 장면** 관찰
5. 일부러 틀린 데이터를 넣고 → subagent가 잡아내는지 확인 (하이라이트 순간!)

### 관찰 포인트
- 메인 AI가 `Agent(...)` 를 호출하는 순간 — "지금 검수 요리사를 부르고 있어요"
- subagent는 **TaskA_output.md 파일만 보고** 검토한다 = 깨끗한 작업대
- Task C의 "미진하면 A부터 재시작" = 품질 루프

## 퀴즈

**Q1. 검증을 별도 AI에게 맡기는 가장 큰 이유는?**
① 더 싸서 / ② 깨끗한 컨텍스트라 자기 결론에 물들지 않아서 ✅ / ③ 더 빨라서 / ④ 멋있어서

**Q2. Task A→B→C 패턴에서 subagent에게 결과를 전달하는 좋은 방법은?**
① 텔레파시 / ② 출력 파일(TaskA_output.md) 경로를 prompt에 적어준다 ✅ / ③ 전달 안 해도 안다 / ④ 스크린샷

**Q3. subagent가 "고객명 누락"을 잡아내려면?**
① 알아서 잡는다 / ② prompt에 검증 기준을 글로 적어야 한다 ✅ / ③ 모델을 바꾼다 / ④ 불가능
