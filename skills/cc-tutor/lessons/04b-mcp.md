# L4.5. MCP 맛보기 — AI에게 도구 달아주기

## 이 레슨의 목표
- MCP가 뭔지 비유로 말할 수 있다 (Skill=업무지시서, Agent=독립적으로 일하는 사람, **MCP=통로**)
- 커넥터 설치가 **어렵지 않다**는 걸 체감한다 — "설치해줘" 한마디면 됨
- 스킬에서 MCP를 쓰는 법 = "○○ MCP의 △△ 도구를 써라" **한 줄**

---

## 개념 (짧게)

지금까지 AI는 내 컴퓨터 파일만 만졌다. **바깥세상**(Gmail, 노션, 법령DB, 캘린더)을 만지려면 도구가 필요 — 그 연결 표준이 **MCP**.

비유: 업무지시서(Skill)에 "A사 DB에서 찾아와"라고 적어도 **바깥으로 통하는 통로(MCP)가 없으면** 못 한다. 업무지시서와 통로는 세트.

## 설치 — 겁먹을 것 없음, 딱 2종류

| 종류 | 설치 방법 | 예시 |
|---|---|---|
| **공식 커넥터** | claude.ai 설정에서 **클릭 설치** → 로그인(인증)만 하면 끝 | Gmail, Google Drive, Notion, 캘린더 |
| **비공식(커뮤니티) MCP** | Claude에게 **"○○ MCP 설치해줘"** 라고 시키면 됨 → 인증정보(API키 등)만 넣어주면 끝 | 세법DB, 녹취록, 브라우저 자동화 등 |

> 핵심 멘트: "어려울 게 없습니다. ① 설치해줘 ② 인증 ③ 끝. 설치 과정 자체를 Claude에게 시켜도 됩니다 — '○○ MCP 설치해줘' 라고 치면 알아서 합니다." (예: 세무사라면 법령DB MCP, 마케터라면 노션 MCP)

확인 방법: `/mcp` 를 치면 지금 연결된 도구 목록이 보인다.

## 스킬에서 도구 쓰는 법 — 코딩 아님, 문장 한 줄

SKILL.md에 이렇게 적으면 끝 (예시 — 세법DB MCP를 연결한 경우):

```markdown
## 할 일
1. ○○ MCP의 law_search 도구로 조문을 검색한다.
2. 찾은 조문 번호를 law_detail 도구로 한 번 더 확인한다.  ← 환각 방지!
```

도구 이름을 정확히 적을수록 안정적으로 작동한다. **내장 도구(WebFetch 등)도 같은 방식**으로 한 줄이면 된다.

## 체험 시나리오 — docs-lookup (누구나 가능, 설치만 하면 됨)
- 설치: `cp -r ${CLAUDE_PLUGIN_ROOT}/skills/cc-tutor/examples/docs-lookup ~/.claude/skills/docs-lookup`
1. `/docs-lookup` 발동 → "Claude Code에서 스킬은 어디에 저장해?"
2. 관찰 포인트 3개:
   - AI가 **도구(WebFetch)를 호출**해 공식 문서를 직접 읽는 장면 — "지금 바깥세상으로 통하는 통로를 쓰고 있어요!"
   - 답변에 **근거(문서 URL + 인용)** 가 붙는 것
   - 원문을 **한 번 더 확인** — "AI는 그럴듯한 옵션명을 지어내기도 해요(환각). 원본 재조회가 검증" ← L4 subagent 검증과 같은 철학!
3. docs-lookup의 SKILL.md를 열어 해부: 도구 사용 지시가 **문장 한 줄**이라는 것 확인
4. 마무리 멘트: "당신 업무의 MCP(법령DB, 사내시스템...)를 연결하면 **똑같은 한 줄**로 그 도구를 쓰게 됩니다."

## 퀴즈

**Q1. MCP를 비유하면?**
① 업무지시서 / ② 통로 ✅ / ③ 독립적으로 일하는 사람 / ④ 회사 건물

**Q2. 비공식 MCP 설치, 가장 쉬운 방법은?**
① 코드를 직접 짠다 / ② Claude에게 "설치해줘"라고 시킨다 ✅ / ③ 설치 불가 / ④ 포맷 후 재설치

**Q3. 스킬에서 MCP를 쓰게 하려면?**
① 파이썬 코드 / ② SKILL.md에 "○○ MCP의 △△ 도구를 써라" 한 줄 ✅ / ③ 폴더명을 mcp로 / ④ 못 씀
