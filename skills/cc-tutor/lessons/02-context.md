# L2. 컨텍스트의 한계를 알다

## 이 레슨의 목표
- 토큰과 컨텍스트 윈도우(작업대)가 뭔지 구체적인 숫자로 안다
- "AI는 기억하지 않는다, 매번 다시 읽는다"를 이해한다
- Lost in the Middle 문제를 알고 → "중요한 건 앞/뒤, 길어지면 새 세션" 습관을 갖는다
- 큰 지식은 파일로 분리하고 필요할 때만 읽게 한다

---

## 개념 1: 토큰 — AI가 글을 읽는 단위

AI는 글자를 그대로 읽지 않고, **텍스트를 잘게 쪼갠 조각(토큰)** 단위로 읽어요.

- 영어: 단어 하나 ≈ 1~2토큰 ("apple" = 1토큰)
- 한글: 글자 1~2자 ≈ 1토큰 ("부가가치세" ≈ 3~4토큰)
- 비유: **쌀알**이라고 생각하면 됩니다. 문서를 밥솥에 넣으려면 쌀알 단위로 세는 것.

> 튜터 멘트: "토큰은 그냥 'AI가 읽는 글자 묶음'이에요. 정확히 외울 필요 없고, '문서 양을 재는 단위구나' 정도면 충분합니다."

## 개념 2: 컨텍스트 윈도우 — 작업대의 크기

AI가 **한 번에 올려놓고 볼 수 있는 토큰의 최대량**이 정해져 있어요. 이게 컨텍스트 윈도우(= 작업대 크기).

아래 표는 **작성 시점(2026-06) 기준 예시**입니다. 모델은 계속 새로 나오니 숫자보다 "모델마다 작업대 크기가 다르다"는 개념이 핵심:

| 모델 | 한마디로 | 작업대 크기 | 체감 분량 |
|---|---|---|---|
| **Opus** (대형) | 가장 똑똑한 모델 | **1M (100만) 토큰** | 📚 **해리포터 시리즈 전권(7권)급** |
| **Sonnet** (중형) | 속도·지능 균형형 | 1M 토큰 | 동일 |
| **Haiku** (소형) | 가장 빠르고 저렴 | 200K (20만) 토큰 | 📕 두꺼운 책 1~2권 |

- 모델명 뒤 숫자(4.6 → 4.7 → 4.8)는 그냥 **버전**. 높을수록 최신.
- `[1m]` 표시 = 컨텍스트 윈도우 1,000,000 토큰이라는 뜻.

> 🔄 튜터 지침: 학습자가 "지금 최신 모델이 뭐야?", "정확한 수치는?"을 물으면 외운 값으로 답하지 말고, **WebFetch로 공식 문서(https://docs.claude.com/en/docs/about-claude/models/overview)를 확인한 뒤** 답한다. "최신 정보는 항상 원본을 확인한다" — 이것 자체가 L4에서 배울 환각 방지 습관의 예고편!

> 튜터 멘트: "100만 토큰이면 엄청 크죠? '그럼 한계 걱정 없겠네'라고 생각하기 쉬운데... 다음 개념이 반전입니다."

## 개념 3: AI는 '기억'하지 않는다 — 매번 처음부터 다시 읽는다

많이들 오해하는 부분: AI가 대화를 "기억하고 있다"고 생각하지만, 사실은 —

**매 질문마다, 지금까지의 대화 전체를 처음부터 다시 읽습니다.**

```
1번째 질문: [질문1] 읽고 답변
2번째 질문: [질문1 + 답변1 + 질문2] 전부 다시 읽고 답변
3번째 질문: [질문1 + 답변1 + 질문2 + 답변2 + 질문3] 전부 다시...
```

그래서 생기는 문제 2가지:

1. **작업대가 점점 차오른다** — 대화가 길어질수록 읽을 게 쌓이고, 느려지고, 한계에 가까워짐
2. **자기가 한 말에 물든다** — 앞에서 내린 결론을 계속 다시 읽으니까, 틀린 결론도 "일관되게" 유지하려는 경향 (사람의 확증편향과 비슷). 이게 L4에서 배울 **"검증은 깨끗한 새 AI에게"** 의 이유!

## 개념 4: Lost in the Middle — 중간은 잊힌다

작업대가 크더라도, **위치에 따라 기억력이 다릅니다.**

🧠 3시간 회의가 끝나면 뭐가 기억나나요?
→ 시작할 때 안건 + 마지막 결론.
→ 중간 2시간의 논의는 가물가물.
**AI도 똑같습니다!**

```
기억력
높음 ●                          ●
     │ ●                      ●
     │   ●                  ●
낮음 │     ● ● ● ● ● ● ● ●
     └───────────────────────
      맨앞     중간(잊힘!)   맨뒤
     "첫 지시"              "방금 한 말"
```

⚡ **그래서 이렇게 하세요 (행동수칙 3줄):**
1. **중요한 규칙은 맨 앞이나 맨 뒤에** — 긴 지시문 한가운데 묻지 말 것
2. **대화가 길어지면 새 세션으로** — 작업대를 비우고 새로 시작
3. **검증은 깨끗한 새 AI(subagent)에게** — L4에서 배움

## 개념 5: 그래서 — 큰 지식은 파일로 분리한다

위 한계 때문에 나온 해법: **SKILL.md에 다 안 넣고, 필요한 것만 꺼내 읽게 한다.**

```
memo-clean/
├── SKILL.md          ← 업무지시서 (얇게) — "먼저 references/용어집.md를 읽어"
└── references/
    └── 용어집.md      ← 지식 (두꺼워도 OK, 필요할 때만 로드)
```

### ⚠️ 용어 주의 — "references"라는 말에 헷갈리지 말 것

학습자가 자주 묻는 질문 (실제 질문에서 나옴):

**Q. "references도 그냥 스킬의 일부 아니야?"**
→ **맞습니다!** 전부 그냥 스킬 폴더 안의 파일이에요. `references/`, `templates/` 는 **그냥 폴더 이름 관례**일 뿐, 특별한 기능이 아닙니다. 폴더명은 아무거나 해도 됩니다. 작동 원리는 단 하나: **SKILL.md 안에 "이 파일을 읽어라"라고 적었기 때문에 읽는 것.**

**Q. "스킬이 작으면 그냥 한 파일에 다 때려넣는 게 낫지 않아?"**
→ **맞습니다!** 판단 기준:
- SKILL.md가 **짧으면 (~100줄 이하)** → 한 파일에 다 넣기 (mail-polish 방식) ✅
- **길어지면** → 분리 (memo-clean 방식) ✅
- 파일 분리 시 경로를 못 찾을 걱정: SKILL.md에 `references/용어집.md` 처럼 **경로를 정확히 적으면** 문제없음

> 튜터 지침: 초보 단계에서는 "references"라는 용어를 먼저 꺼내지 말 것. "SKILL.md + 같이 들어있는 파일"이라고만 설명하고, 학습자가 구조를 본 후에 "이런 걸 관례상 references 폴더라고 불러요" 순서로.

## 체험 시나리오
1. `/memo-clean` 발동 → 휘갈긴 메모 정리 체험 ("부가세→부가가치세" 변환 관찰)
2. `references/용어집.md`를 열어서 "아, 표에 적힌 대로 바꿨구나" 확인
3. SKILL.md의 "먼저 references/용어집.md를 읽는다" 지시 줄 확인 — **읽으라고 적어야 읽는다**
