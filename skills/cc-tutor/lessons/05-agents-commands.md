# L5. 실무사례 투어 — 진짜 스킬은 이렇게 생겼다 (+ Agent 개념)

## 학습 목표
- 지금까지 배운 부품(업무지시서/지식/양식/검증/통로)이 **실무 스킬 안에서 전부 조립된 모습**을 본다.
- **Agent = 독립적으로 일하는 사람**, subagent(보조 일꾼)의 격리·병렬 개념을 사례로 이해한다.
- 졸업과제(L6) 아이디어를 줍는다.

## 진행 방식 (튜터 노트)
- 이 레슨은 **만들기 없음. 박물관 투어다.** 사례 3개를 차례로 열고 "어느 레슨 부품이 보이나요?" 문답으로 진행.
- 한 사례당 5분 이내. 어렵다는 신호가 오면 "구조만 보면 됩니다 — 부품은 전부 배운 것"이라고 안심시킬 것.
- 투어 순서 (쉬움 → 화려함):

### 🏛 1관: cases/fee-invoice.md — 가장 단순한 실무 스킬
- 보여줄 것: 청구서 docx의 값만 바꿔치기 — **L1(업무지시서)+L3(양식 고정)** 그대로.
- 문답: "이 스킬에서 templates에 해당하는 건 뭘까요?" / "우리가 굴린 notice-maker와 뭐가 같죠?"
- 메시지: **실무 스킬도 결국 배운 부품 그대로다.**

### 🏛 2관: cases/jongsose.md — 검증이 목숨인 스킬
- 보여줄 것: 종소세 안내문 — 생성하는 쪽과 **검증(verifier)하는 쪽이 분리**, 검증 실패 시 **저장 보류(멈춤)**.
- 문답: "왜 만들고 나서 멈추는 단계를 뒀을까요?" (틀린 안내문이 거래처에 나가면 사고 → L3 바닥의 실전판)
- 여기서 Agent 개념 투입: 검사를 **별도의 깐깐한 일꾼**에게 맡기면? → subagent.

### 🏛 3관: cases/realestate.md — 일꾼을 여럿 부리는 스킬
- 보여줄 것: 서울 25개구 조사를 **agent 여러 명이 나눠서 동시에** → 메인이 취합.
- 문답: "왜 한 명이 다 안 하고 쪼갰을까요?" (작업대=Context Window에 다 안 올라감 + 속도)
- subagent 핵심 성질 2개만: **격리**(내가 준 것만 안다) / **병렬**(여럿 동시에).

### Agent 개념 정리 (투어 마무리, 표 하나로)
| | Skill | Agent |
|---|---|---|
| 비유 | 업무지시서 | 독립적으로 일하는 사람 |
| 단위 | 능력 (시키면 잘함) | 위임 (알아서 판단) |
| 위치 | `~/.claude/skills/` | `~/.claude/agents/` |
- 성장 경로: **스킬로 쓰다가 "항상 켜두고 일 시키는" 수준이 되면 agent로 승격.** (지금 이 cc-tutor가 바로 agent!)
- command는 한 줄만: "자주 쓰는 스킬 호출의 단축키" 정도로 소개하고 끝.

## 졸업과제 씨앗 뿌리기 (투어 출구에서)
- "오늘 본 사례 중 **내 업무랑 가장 닮은 건** 어떤 거였나요?"
- "내일 출근하면 또 해야 하는 반복 작업 하나만 꼽는다면?" → 답을 dashboard 학습자 메모에 기록 (L6 재료).

## 퀴즈

**Q1. "알아서 판단하며 일하는 위임 단위"는?**
- ① Skill / ② Command / ③ Agent ✅ / ④ Template
해설: Skill은 능력 단위(업무지시서), Agent는 위임 단위(독립적으로 일하는 사람).

**Q2. 종소세 스킬에서 검증(verifier)이 실패하면?**
- ① 그래도 일단 저장한다 / ② 저장을 보류하고 멈춘다 ✅ / ③ 검증을 끄고 재시도 / ④ 다른 거래처로 넘어간다
해설: 틀린 안내문이 나가는 것보다 멈추는 게 낫다 — 바닥 설계의 실전판.

**Q3. 부동산 스킬이 조사를 여러 agent로 쪼갠 가장 큰 이유는?**
- ① 멋있어 보여서 / ② 한 작업대(컨텍스트)에 다 안 올라가는 양이라 쪼개서 병렬 처리 ✅ / ③ agent가 많을수록 무조건 정확해서 / ④ MCP가 하나만 응답해서
해설: Context Window 한계 + 속도. 쪼개기와 취합이 하니스 설계.

## 마무리 3줄 요약 포인트
- 실무 스킬 = 배운 부품(업무지시서/지식/양식/검증/통로)의 조립품. 새로운 건 없었다!
- Agent=독립적으로 일하는 사람(위임), subagent=격리된 보조 일꾼(검증·병렬).
- 다음 시간: **졸업과제** — 드디어 자기만의 스킬을 처음부터 직접 만든다.
