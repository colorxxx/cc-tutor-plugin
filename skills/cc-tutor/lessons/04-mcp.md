# L4. MCP 맛보기 — 바깥세상으로 통하는 통로를 놓다

## 학습 목표
- MCP가 뭔지 비유로 설명할 수 있다 ("AI와 바깥세상을 잇는 만능 어댑터").
- 스킬 본문에 "○○ MCP의 △△ 도구를 써라" 한 줄이면 도구가 연결된다는 것을 체험한다.
- MCP 결과도 **재확인**해야 한다는 습관(환각 방지)을 본다.

## 개념 (짧게 — 설치 이론은 최소화)
- 지금까지 AI는 내 컴퓨터 파일만 만졌다. **바깥세상**(메일, 노션, 법령 DB)을 만지려면 도구가 필요 — 그 연결 표준이 **MCP = 통로 (바깥세상으로 통하는 길)**.
- 비유 완성: 업무지시서에 "A사 DB에서 찾아와"라고 적어도 **바깥으로 통하는 통로(MCP)가 없으면** 못 한다. 업무지시서(Skill)와 통로(MCP)는 세트.
- 등록 범위는 한 줄만: "**user로 깔면 모든 프로젝트에서** 쓴다" (local=이 폴더만, project=팀 공유). 깊이는 안 들어간다.
- 설치도 깊이 안 간다: claude.ai 커넥터 클릭 설치가 있다는 것 + **"설치 과정 자체를 Claude에게 시켜도 된다"** 꿀팁만.

## 설치 (복사 한 줄)
```
cp -r ${CLAUDE_PLUGIN_ROOT}/skills/cc-tutor/examples/docs-lookup ~/.claude/skills/docs-lookup
```
- docs-lookup은 내장 WebFetch 도구를 쓰므로 별도 MCP 설치 없이 누구나 체험 가능. 학습자에게 이미 연결된 MCP가 있다면(`/mcp`로 확인) 그 도구로 같은 패턴을 보여줘도 좋다.

## 체험 시나리오 (메인)
1. `/docs-lookup` 발동 → "Claude Code에서 스킬은 어디에 저장해?" 같은 질문.
2. 관찰 포인트 3개:
   - AI가 **도구를 호출**하는 장면 (WebFetch) — "지금 바깥세상으로 통하는 통로를 쓰고 있어요!"
   - 답변에 **근거(문서 URL + 인용)** 가 붙는 것
   - 원문을 **한 번 더 확인**하는 것 — "AI는 가끔 그럴듯한 옵션명을 지어내요(환각). 그래서 원본 재조회가 검증이에요" ← L3 바닥의 진화형!
3. SKILL.md를 열어 해부: "○○ 도구로 △△하라" **문장 한 줄**이 전부라는 것 — 코딩 없음. 업무 MCP를 연결해도 똑같은 한 줄.

## 한 줄 수정 (유일한 만들기)
- 답변 양식에 자기가 원하는 줄 하나 추가 (예: "**실무 한마디**: 실무자가 주의할 점 1줄").
- 재발동 → 새 양식 반영 확인.

## 퀴즈

**Q1. MCP를 비유하면?**
- ① 업무지시서 / ② 통로 ✅ / ③ 독립적으로 일하는 사람 / ④ 회사 건물
해설: AI가 외부 세계로 나가는 통로. 업무지시서(Skill)와 세트.

**Q2. 스킬에서 MCP를 쓰게 하는 방법은?**
- ① 파이썬 코드를 짠다 / ② SKILL.md에 "○○ MCP의 △△ 도구를 써라"라고 적는다 ✅ / ③ MCP는 스킬에서 못 쓴다 / ④ 폴더 이름을 mcp로 바꾼다
해설: 지시문 한 줄이면 끝. 도구 이름을 정확히 적을수록 안정적.

**Q3. docs-lookup이 찾은 내용을 원문으로 한 번 더 확인하는 이유는?**
- ① 조회 횟수를 늘리려고 / ② AI가 지어낸 내용(환각)을 걸러내려고 ✅ / ③ 서버를 깨우려고 / ④ 답변을 길게 하려고
해설: 원본 재조회가 가장 확실한 바닥(검증)이다.

## 마무리 3줄 요약 포인트
- MCP = 바깥으로 통하는 통로(만능 어댑터). 업무지시서와 통로는 세트.
- 스킬에서 쓰는 법 = "○○ MCP로 …하라" 한 줄.
- 다음 시간: **실무사례 투어** — 진짜 회계법인에서 굴러가는 스킬은 어떻게 생겼는지 구경 + 독립적으로 일하는 사람(Agent) 개념.
