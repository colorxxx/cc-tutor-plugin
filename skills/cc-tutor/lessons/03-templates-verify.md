# L3. 양식+검증 체험 — 틀리면 멈추는 스킬

## 학습 목표
- templates/ 로 출력 **양식을 고정**하는 구조를 체험으로 안다.
- **검증 체크리스트**가 실수를 잡아내는 장면을 직접 본다.
- "좋은 스킬 3요소": 프롬프트=천장 / 검증=바닥 / description=문 을 비유로 안다.

## 개념 (짧게)
- references가 "알아야 할 것"이라면 **templates는 "결과물이 닮아야 할 모양"**. 같은 원리(폴더에 번들).
- **좋은 스킬 3요소**:
  - 정확한 프롬프트 = **천장** (최고 품질을 올림)
  - 결과 검증 = **바닥** (최악의 사고를 막음) — 세무처럼 숫자 틀리면 안 되는 업무에선 바닥이 더 중요!
  - description = **문** (안 불려나오면 무용지물 — L1 복습)
- 검증의 기본형: 끝에 체크리스트 + **"실패하면 수정하고 다시 검증하라"** 한 줄.

## 설치 (복사 한 줄)
```
cp -r ${CLAUDE_PLUGIN_ROOT}/skills/cc-tutor/examples/notice-maker ~/.claude/skills/notice-maker
```

## 체험 시나리오 (메인) — 2번 발동이 핵심
1. **일부러 정보를 빠뜨리고** 발동: `/notice-maker` → "한빛상사에 부가세 증빙자료 요청 안내문 만들어줘"
   - 관찰 포인트: 바로 안 만들고 **"마감일과 담당자를 알려달라"고 되묻는다!** ← 빠지면 만들지 말라는 지시가 작동한 것.
2. 정보를 다 주고 완성: "마감일 6월 20일, 담당자 김OO"
   - 관찰 포인트: 결과가 templates/안내문양식.md와 **토씨까지 같은 모양**, {중괄호}는 전부 치환됨.
3. SKILL.md를 같이 열어 해부: ① "빠졌으면 물어봐라" ② "양식 그대로" ③ 검증 체크리스트 3개 + "실패 시 재검증" — 각 지시와 방금 본 동작을 짝짓기.
- 깨달음 멘트: "방금 스킬이 **두 번 일했어요**. 한 번은 만들고, 한 번은 검사하고. 검사하는 한 줄이 사고를 막는 **바닥**입니다."

## 한 줄 수정 (유일한 만들기)
- 양식 파일의 문구 하나를 자기 사무실 말투로 고치게 한다 (예: 맺음말 → "늘 감사드립니다.").
- 재발동 → 바뀐 양식 그대로 나오는 것 확인.
- (여유 시) 검증 체크리스트에 4번째 항목 추가: "□ 전화번호가 들어갔는가?" → 어떻게 동작이 달라질지 예측해보게 하고 실험.

## 퀴즈

**Q1. "좋은 스킬 3요소"에서 결과 검증의 역할은?**
- ① 천장 — 최고 품질을 올린다 / ② 바닥 — 최악의 사고를 막는다 ✅ / ③ 문 — 발동을 돕는다 / ④ 지붕 — 비용을 줄인다
해설: 프롬프트가 천장, 검증이 바닥. 실무에선 사고를 막는 바닥이 먼저다.

**Q2. notice-maker가 마감일 없이 요청받았을 때 한 행동은?**
- ① 마감일을 대충 지어냈다 / ② 만들지 않고 먼저 물어봤다 ✅ / ③ 마감일 칸을 비웠다 / ④ 오류로 종료됐다
해설: "빠지면 만들지 말고 물어봐라" — 지어내는 것보다 멈추고 묻는 게 안전. 이것도 바닥 설계.

**Q3. 결과물 양식을 통일하는 가장 좋은 방법은?**
- ① 매번 말로 양식 설명 / ② templates/ 양식 파일 + "이대로 따르라" 지시 ✅ / ③ AI 재량에 맡기기 / ④ 받은 뒤 수작업 수정
해설: 양식은 파일로 고정, {치환변수}만 바꾸게 한다.

## 마무리 3줄 요약 포인트

- 검증 체크리스트 + "실패 시 재검증" 한 줄이 품질의 바닥을 깐다.
