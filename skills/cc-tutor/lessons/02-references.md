# L2. 지식 끼워넣기 체험 — references와 용어집

## 학습 목표
- references/ 폴더로 **암묵지**를 스킬에 번들하는 구조를 체험으로 안다.
- **progressive disclosure**(필요할 때만 꺼내 읽기)를 한 줄로 설명할 수 있다.
- 용어집에 한 줄을 추가해서 **출력이 바뀌는 것**을 본다.

## 개념 (짧게)
- L0 복습: "AI에게는 암묵지가 필요하다" → 그 답: **스킬 폴더에 지식 파일을 같이 넣는다.**
```
memo-clean/
├── SKILL.md          ← 업무지시서 (얇게)
└── references/
    └── 용어집.md      ← 나만 아는 지식 (두꺼워도 OK)
```
- **progressive disclosure**: 평소엔 이름+description만 보임 → 발동하면 본문 로드 → references는 **필요한 순간에만** 읽음. 그래서 지식 파일은 두꺼워도 공짜에 가깝다. **"업무지시서는 얇게, 백과사전은 두껍게."**

## 설치 (복사 한 줄)
```
cp -r ${CLAUDE_PLUGIN_ROOT}/skills/cc-tutor/examples/memo-clean ~/.claude/skills/memo-clean
```

## 체험 시나리오 (메인)
1. `/memo-clean` 발동 → 이런 휘갈긴 메모를 던지게 한다:
   > "한빛상사 전화옴. 부가세 기한 묻길래 25일이라함. 원징 자료요청 내일까지 달라했음 1500000 입금예정"
2. 결과 관찰: "부가세→부가가치세", "원징→원천징수", "1,500,000원" 으로 바뀌어 있다!
3. **"AI가 어떻게 알았지?"** → `references/용어집.md`를 같이 연다 → 표와 출력을 짝지어 보여준다.
4. SKILL.md의 "먼저 references/용어집.md를 읽는다" 지시 줄도 보여주기 — **읽으라고 적어야 읽는다.**

## 한 줄 수정 (유일한 만들기)
- 용어집 표에 **자기가 실제 쓰는 줄임말 1개**를 추가하게 한다 (예: "법인결산 → 법인 결산 업무").
- 그 줄임말이 든 메모로 재발동 → 새 규칙이 반영되는 것 확인.
- 포인트 멘트: "방금 하신 게 바로 **암묵지 주입**이에요. 코딩 아니고 표에 한 줄 쓰기였죠?"

## 퀴즈

**Q1. 분량 많은 업무 지식을 스킬에 넣는 가장 좋은 방법은?**
- ① SKILL.md 본문에 전부 붙여넣기 / ② references/ 파일로 분리하고 필요할 때 읽게 하기 ✅ / ③ 매번 대화창에 복사 / ④ description에 넣기
해설: 업무지시서는 얇게, 백과사전은 두껍게.

**Q2. progressive disclosure 설명으로 맞는 것은?**
- ① 스킬을 단계적으로 유료 결제 / ② 평소엔 이름+설명만, 본문·참조파일은 필요할 때만 로드 ✅ / ③ 스킬을 점점 길게 쓰기 / ④ 시작할 때 모든 파일을 한꺼번에 읽기
해설: 단계적 공개. 그래서 references가 커도 작업대(컨텍스트)가 안 좁아진다.

**Q3. 용어집에 한 줄을 추가했더니 출력이 바뀌었다. 이걸 한마디로?**
- ① 모델 업그레이드 / ② 암묵지 주입 ✅ / ③ 하드웨어 교체 / ④ 우연
해설: 모델은 그대로인데 내 지식이 들어가서 결과가 좋아졌다 — 이게 L0의 "암묵지" 실체.

## 마무리 3줄 요약 포인트
- 암묵지는 references/ 파일로 번들. "읽으라"고 지시 한 줄이면 끝.
- progressive disclosure 덕에 지식 파일은 두꺼워도 부담 없다.
- 다음 시간: 출력 **양식(템플릿)을 고정**하고 **검증이 오류를 잡아내는** preset을 굴려본다.
