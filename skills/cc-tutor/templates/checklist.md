# 체험 확인 체크리스트 (튜터용)

각 레슨 마무리 전에 항목별로 확인하고 결과를 학습자에게 보여준다.
통과 기준: 필수(✱) 전부. L1~L4는 체험이 목적이므로 **관찰만 해도 통과** — 막히면 튜터가 시연해주고 넘어간다.

## L0 오리엔테이션
- [ ] ✱ 자연어 지시로 파일 생성·수정 체험
- [ ] 암묵지/하니스를 자기 말로 설명
- [ ] Shift+Tab 모드 전환 구경

## L1 스킬 발동 (mail-polish)
- [ ] ✱ preset 복사 + `/mail-polish` 발동 성공
- [ ] ✱ 규칙(SKILL.md)과 출력의 1:1 대응을 확인함
- [ ] ✱ 서명 블록을 자기 것으로 수정 + 재발동으로 변화 확인
- [ ] 폴더명=명령어 관계를 설명

## L2 지식 끼워넣기 (memo-clean)
- [ ] ✱ 발동 → 용어집이 출력에 반영되는 것 확인 (부가세→부가가치세 등)
- [ ] ✱ 용어집에 자기 줄임말 1줄 추가 + 재발동으로 반영 확인
- [ ] progressive disclosure를 한 줄로 설명

## L3 양식+검증 (notice-maker)
- [ ] ✱ 정보 빠진 요청 → 되묻는 동작 확인
- [ ] ✱ 완성본이 양식과 동일 + {중괄호} 전부 치환 확인
- [ ] ✱ 양식 문구 1곳 수정 + 재발동으로 반영 확인
- [ ] 천장/바닥/문 구분 설명

## L4 MCP (docs-lookup)
- [ ] ✱ 발동 → 도구(WebFetch) 호출 장면 관찰 (없으면 fallback 경로로 OK)
- [ ] ✱ "재확인(환각 방지)"이 왜 필요한지 설명
- [ ] 답변 양식에 한 줄 추가 + 재발동

## L5 실무사례 투어
- [ ] ✱ 사례 3개에서 배운 부품(레시피/양식/검증/도구)을 찾아 말함
- [ ] ✱ skill vs agent 차이를 비유로 설명
- [ ] 졸업과제 후보 1개를 dashboard에 기록

## L6 졸업과제
- [ ] ✱ 자기 업무 스킬 1개 완성 + 실데이터 시연
- [ ] ✱ 검증(바닥)이 실제 오류를 잡음 (정보 빠진 입력 테스트)
- [ ] 설계 5문항(입력출력/암묵지/양식/도구/바닥)에 답함
- [ ] ✱ 졸업시험 4/5 이상
